# Vivarium

A live C# scripting environment exposed as an [MCP](https://modelcontextprotocol.io) server.
Vivarium gives AI assistants — and adventurous humans — a persistent, stateful
workspace where code definitions accumulate across interactions and survive
restarts.

Think of it as a Smalltalk image for the age of AI: an always-on programming
session where you define classes and functions, run ad-hoc queries, and build up
a personal library of tools — all through a clean MCP tool interface.

## Why?

AI coding assistants spend enormous effort in write → run → read-output → rewrite
loops. Each cycle burns tokens, costs money, and loses context. Vivarium
short-circuits this: the AI defines a utility once, and it stays alive in the
session — ready to call, inspect, and compose with other utilities — for as long
as the server runs. Definitions are persisted as plain `.cs` files, so they
reload automatically on restart.

## Features

- **Live scripting session** — incremental Roslyn scripting engine where each
  `eval` builds on prior state (variables, types, usings persist).
- **Persistent definitions** — `define` saves C# source files under
  `.vivarium/src/` with metadata headers (`@description`, `@depends`,
  `@exports`). Files reload on server restart.
- **Auto-exports** — public symbols are automatically extracted via Roslyn at
  define-time and stored as `@exports:` metadata for discoverability.
- **Dependency ordering** — files declare `@depends:` and are loaded in
  topological order.
- **10 MCP tools** — `eval`, `define`, `list`, `view`, `delete`, `inspect`,
  `inspect_var`, `reset`, `search`, `catalog`.
- **Full .NET power** — `#r` references, async/await, LINQ, System.Text.Json,
  HttpClient, Regex — the entire BCL is available.

## Quick start

### Prerequisites

- [.NET 10 SDK](https://dotnet.microsoft.com/download/dotnet/10.0)

### Build

```bash
git clone https://github.com/sivadeilra/vivarium
cd vivarium
dotnet build src/Vivarium/Vivarium.csproj -c Release
```

The binary lands at `src/Vivarium/bin/Release/net10.0/Vivarium.exe`.

### Install as an MCP server

Add to your VS Code `.vscode/mcp.json` (or the equivalent for your MCP host):

```json
{
    "servers": {
        "vivarium": {
            "command": "/path/to/Vivarium.exe",
            "type": "stdio"
        }
    }
}
```

Or set the working directory and data root explicitly:

```json
{
    "servers": {
        "vivarium": {
            "command": "/path/to/Vivarium.exe",
            "args": ["--root", "/path/to/.vivarium"],
            "type": "stdio"
        }
    }
}
```

### Data directory

Vivarium stores definitions under a `.vivarium/src/` directory.
Resolution order:

1. `--root` command-line argument
2. `VIVARIUM_ROOT` environment variable
3. `$CWD/.vivarium`

## MCP tools

| Tool | Description |
|---|---|
| `vivarium_eval` | Execute arbitrary C# in the live session |
| `vivarium_define` | Create/update a persistent source file and load it |
| `vivarium_list` | List all definition files with descriptions |
| `vivarium_view` | View the full source of a definition file |
| `vivarium_delete` | Delete a definition file (warns about dependents) |
| `vivarium_inspect` | List all live variables with types and values |
| `vivarium_inspect_var` | Deep-inspect a variable: type, value, members |
| `vivarium_reset` | Clear session state and reload all files from disk |
| `vivarium_search` | Search file names and source code |
| `vivarium_catalog` | Rich library view: descriptions, exports, dependencies |

## File format

Definition files are plain C# with metadata in header comments:

```csharp
//@VIVARIUM@
//@description: Math utilities
//@depends: Utils/Helpers.cs
//@exports: MathUtils, MathUtils.Factorial(1), MathUtils.Gcd(2)

public static class MathUtils
{
    public static long Factorial(int n) =>
        n <= 1 ? 1 : n * Factorial(n - 1);

    public static int Gcd(int a, int b) =>
        b == 0 ? a : Gcd(b, a % b);
}
```

The `//@VIVARIUM@` marker is auto-added if you omit it. The `@exports:` line is
auto-generated by Roslyn analysis at define-time.

## Architecture

```
src/Vivarium/
  Program.cs           — Host setup, DI, MCP stdio transport
  ScriptingEngine.cs   — Roslyn CSharpScript wrapper with incremental state
  FileStore.cs         — File CRUD + metadata parsing for .vivarium/src/
  BootstrapLoader.cs   — Startup loader with topological dependency sort
  SymbolExtractor.cs   — Roslyn-based public symbol extraction
  Tools.cs             — All MCP tool handlers
```
